## 问题一：LLM 使用分析

### 当前 LLM 调用全景

| 环节 | LLM 用途 | 调用频率 | 能否替代 | 影响评估 |
|------|----------|----------|----------|----------|
| **新闻/Reddit 情感分析** | 对每条新闻/帖子打 [-1,1] 分 | 每条内容 1 次 | ✅ 规则替代 | 低：规则情感词典(关键词匹配)可达 70-80% 精度 |
| **动态阈值计算** | 根据市场环境调整置信度阈值 | 每个信号 1 次 | ✅ 规则替代 | 中：可用启发式规则(集中度+VIX+交易疲劳)替代 |
| **Signal Processor** | 从分析报告提取 BUY/HOLD/SELL | 每次决策 1 次 | ✅ 规则替代 | 极低：纯关键词提取 |
| **Holding Monitor** | 综合评估持仓风险 | 每轮巡检 1 次 | ⚠️ 部分替代 | 中：规则可替代硬指标检查，但综合判断 LLM 更好 |
| **TradingAgents 分析师** | 市场/社交/新闻/基本面分析 | 每标的 4 次 | ❌ 不建议 | 高：这是系统核心价值 |
| **TradingAgents 牛熊辩论** | Bull↔Bear 辩论(默认3轮) | 每标的 6 次 | ⚠️ 可优化 | 高：可减少轮次(3→2)但不建议完全去掉 |
| **TradingAgents 交易员** | 生成交易计划 | 每标的 1 次 | ❌ 不建议 | 高：核心决策 |
| **TradingAgents 风控辩论** | 三角辩论(默认3轮) | 每标的 6 次 | ⚠️ 可优化 | 高：可减少轮次(3→2) |
| **Reflector** | 回顾历史决策更新记忆 | 每次交易后 | ✅ 批处理 | 低：可改为日终批处理，不影响实时交易 |

### 推荐方案

**立即可替代（规则化，影响小）：**
1. **情感分析 → 规则词典** — 用关键词+正则替代 LLM，速度提升 10x
2. **Signal Processor → 规则提取** — 关键词匹配 BUY/HOLD/SELL
3. **动态阈值 → 启发式规则** — 基于集中度/VIX/交易次数的公式计算
4. **Reflector → 日终批处理** — 不影响实时交易

**建议优化（保留 LLM 但减少调用）：**
5. **TradingAgents 辩论轮次 3→2** — 减少 4 次 LLM 调用/标的
6. **Holding Monitor 增加规则前置过滤** — 规则先筛，LLM 只处理边界情况

**预期效果：**
- LLM 调用次数减少 ~60-70%
- 系统响应速度提升 2-3x
- TradingAgents 核心决策质量不受影响

## 问题二：MSFT 重复买入根因

### 现象
今日 MSFT 被买入 **6 次**（不是 3 次），每次 2 股：
```
11:03 MSFT BUY 2 (score=0.785)
12:32 MSFT BUY 2 (score=0.786)
13:02 MSFT BUY 2 (score=0.786)
17:17 MSFT BUY 2 (score=0.858)
18:48 MSFT BUY 2 (score=0.858)
19:02 MSFT BUY 2 (score=0.858)
```

### 根因
`screener.py` 的去重逻辑 `_get_current_holdings()` 调用 `sync_from_broker()` 同步长桥持仓，
但 **同步的是"已成交持仓"，不包含"今日已下单但未结算"的订单**。

流程：
```
Pipeline Run 1: MSFT 不在持仓 → 通过筛选 → 买入 2 股 → 订单提交但可能未立即成交
Pipeline Run 2: sync_from_broker → MSFT 仍未在持仓(订单处理中) → 再次通过 → 再买 2 股
...重复 6 次
```

### 修复方案
1. **新增"今日已交易标的"检查**：在筛选去重时，不仅检查持仓，还检查今日 trades 表
2. **增加同标的日交易限制**：同一标的日内最多买入 N 次
